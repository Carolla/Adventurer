Development Package

Each use case contains a pre-defined set of files of all the artifacts as it goes through analysis, design, coding, and testing. The artifacts given below are listed in the sequence in which they are written.

Analysis

  • Detailed use case (.doc or .odt)
  • Wireframe of GUI view (.jpg embedded in explanatory.doc or .odt)
  • Sequence diagram of PDC component only (XUML .odg)
  • Class diagram of classes, methods, and data of sequence diagram for this use case only (UML .odg)
  • GUI test case list summary (.xls or .ods format)
  • Integration test case scripts for the use case, executing from input CIV to output CIV (.doc or .odt)
  • GUI test case scripts for the use case, informed by the wireframe context (.doc or .odt)
  • Updated GUI test case summary containing the date each GUI test passed (.xls or .ods format)

    • Design, Coding, and Testing

  • Sequence diagram of all components (XUML .odg)
  • Java code files and associated unit tests (.java>
  • Javadoc comment docs for all code and unit tests (.html)
  • Integration test suite of all JUnit integration tests for this use case so that it can be run as a single regression test <.java>

    • General Iteration Artifacts

  • UOM (Unified Object Model): Conflation of all classes, methods, and data from the detailed sequence diagram and updated class diagram for this use case into a single application-wide class diagram for all use cases currently implemented (UML .odg)
  • Suite of all Integration test suites from each of the use case specific test suites. <.java>
  • User Manual: Document explaining how and why to use the use cases in the applicaiton (.doc>
